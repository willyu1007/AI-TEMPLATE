# AI Maintenance Reports

This folder stores JSON reports generated by `scripts/ai_maintenance.py`. Use them to monitor repository hygiene over time.

## Naming
```
maintenance_YYYYMMDD_HHMMSS.json
```
Example: `maintenance_20251105_103000.json`.

## Report Schema
- `timestamp` - UTC execution time.
- `summary` - counts of total tasks, passed, failed, and warnings.
- `tasks[]` - per-task result with `name`, `status`, `details`, and optional `actions`.

## Inspecting Reports
```bash
ls -lt ai/maintenance_reports/*.json | head -1      # latest file
jq '.' ai/maintenance_reports/<file>.json | less   # pretty print
```

### Quick Trend Script
```bash
python - <<'PY'
import json
from pathlib import Path
reports = sorted(Path('ai/maintenance_reports').glob('*.json'))[-10:]
for path in reports:
    data = json.loads(path.read_text())
    summary = data.get('summary', {})
    print(f"{path.name}: {summary.get('passed', 0)}/{summary.get('total', 0)} passed")
PY
```

## Retention Policy
- Keep at least 30 days of data.
- Never delete reports that contain failures; they are needed for RCA.
- For older success-only reports, delete with a simple `find` command or a housekeeping script.
